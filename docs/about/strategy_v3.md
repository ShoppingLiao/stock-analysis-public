# 🎯 黑飛舞 v3 — TDMD 快評 + 倉位/停損連動

> **這是當前線上版本**(2026-05-06 起)。v1 只跑黑飛舞 / v2 試了寬出場+加碼(失敗)→ v3 改走「分層篩選 + 倉位連動」。
> 完整對比見 [v1/v2/v3 戰績](strategy_compare.md)。

---

## 為什麼要 v3?

v1(現行 dynamic 出場)在無差別觸發下表現最好(期望值 +1.20%/筆)。但實務上有兩個盲點:

1. **「同樣是黑飛舞觸發」品質卻差很多** — 有些是強趨勢 + 大戶吸貨,有些只是技術形態剛好對上
2. **每筆單同樣大小無法放大優勢** — 強訊號跟弱訊號吃同樣資金,稀釋了真正好機會的權重

v3 的思路:**先用快評區分強弱,然後讓部位 + 停損隨分數動態調整**。

## TDMD 四維度快評(0-4 分)

| 維度 | 來源 | 評分邏輯 |
|---|---|---|
| **T** rend(Setup 品質)| 規則 | 距 5MA ≤ 1.5%(+0.5) + 從 D-1 高回 ≥5%(+0.5) |
| **D** emand(量能)| 規則 | 今日 / 20 日均量 比值:≥2 → 1 / 1.3-2 → 0.5 / 否則 0 |
| **M** oney(籌碼)| 規則 + 公開資料 | 法人 + 大戶 + 量能結構 三條合一,取 max ≤ 1 |
| **D** river(題材)| LLM(Stage 2 啟用) | 過去一週有題材 → 1 / 無 → 0 |

**Money 三條合一細節:**

```
法人(0-0.5):
  5 日三大法人淨買 / 20 日均量 ratio
  ≥ 0.5 → 0.5    /   0~0.5 → 0.25   /   < 0(賣超) → 0

大戶(0-0.5,本土主力訊號):
  TDCC 集保戶 1000 張+ 大戶集中度 %
  ≥ 70% → 0.5    /   60-70% → 0.25  /   < 60% → 0
  (再加,若過去 4 週變化 ≥ +1pp → 至少 0.5)

量能結構(0-0.5):
  • 爆量上漲(今日量 ≥ 5 日均量 1.5x 且收紅) → 0.5
  • 連續量縮拉抬(過去 5 日均量 ≤ 20 日均量 0.7x 且 5 日漲幅 > 0) → 0.5
```

## 分數 → 倉位 / 停損 / 停利 連動

| 總分 | 等級 | 部位 | 停損 | 停利 | 行動 |
|---|---|---|---|---|---|
| **4** | 🔴 **主力部位** | 25% 資金 | 寬:**-8 檔** | 寬:**+18%** | **完整 Opus DDMD 自動跑**,放手讓利潤奔跑 |
| 3 | 🟠 中倉 | 8% 資金 | 標準:-5 檔 | 標準:+12% | 不跑完整 DDMD |
| 2 | 🟡 試單 | 3% 資金 | 緊:-3 檔 | 緊:+8% | 不跑完整 DDMD |
| ≤1 | ⚪ 觀察 | 0% | — | — | 不進 |

### 為什麼高分要寬停損?

- 高分(TDMD 4)代表**多重訊號共振**,大戶 + 法人 + 量能 + setup 都對
- 這種標的「短期波動雜訊」較大,但中長期趨勢強
- **緊停損會在雜訊中被洗掉**,失去原本的趨勢機會
- 相反,低分(2)代表「只剩一個訊號還在」,本來就脆弱,**緊停損保命**

## 三層 Pipeline 整合

```
13:00 / 13:15  scan_intraday(黑飛舞觸發 ~5 檔)
        ↓
        TDMD 快評(每檔 0-4 分,規則式 < 1s)
        ↓
        ├ 4 分 → Telegram「主力部位 + 寬停損」+ 等待 Opus DDMD
        ├ 3 分 → Telegram「中倉」
        ├ 2 分 → Telegram「試單」
        └ ≤1 分 → 不進(可能還是會推送但標記「觀察」)

14:50  D-1 預掃 → 對「明日可能形成黑飛舞」的標的跑 TDMD 快評
        → 只對 ≥ 2 分的標的自動跑完整 Opus DDMD(可避免在低品質標的浪費 token)

每週四 18:00  抓 TDCC 集保戶大戶資料(週度 snapshot,累積歷史)
```

## 已知 limitation

1. **大戶資料只能從現在累積**
   TDCC OpenData 只給最新一週 snapshot,沒歷史。
   Week 1 只能看絕對值(集中度 %);Week 2+ 才有變化趨勢可用。

2. **大戶資料週度滯後**
   每週四公布上週三的 snapshot,對日內 trigger 來說滯後 2-7 天。
   但主力動作通常持續多週,仍有預測力。

3. **Driver(LLM 題材)Stage 2 才啟用**
   Stage 1 先跑 3 個規則維度。Driver 維度暫時給「中性 0.5」,不影響其他三維評分。
   這意味著 Stage 1 的最高總分實際是 3.5(而非 4.0)。

4. **券商分點主力沒抓**
   有部分本土主力是「特定券商分點密集進貨」,目前沒納入。
   未來可考慮加 stockq / goodinfo 爬蟲。

## 預期效果(尚未驗證,實戰累積)

| 比較 | v1(現行)| v3(快評 + 倉位連動)|
|---|---|---|
| 每筆吃同樣資金 | ✓ | ❌(動態 0-25%)|
| 看不出強弱 | ✓ | ❌(0-4 分)|
| 追蹤大戶吸貨 | ❌ | ✓(TDCC)|
| 法人共識 | ❌ | ✓(TWSE T86 + TPEx 3IT)|
| 自動 DDMD | 全部跑 | **只對 ≥ 2 分跑** ← 省 60% token |
| 寬鬆停利 | ❌(全 +12%)| 4 分自動拉到 +18% |

**核心假設(待驗證):**
重壓 4 分標的 + 放寬停損 = 平均勝率略降,但**個別大贏單規模放大** = 整體期望值提升。

實戰累積 30-50 筆 4 分案例後,可回看驗證。
